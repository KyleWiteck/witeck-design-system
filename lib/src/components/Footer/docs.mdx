## Basic Usage

```tsx live

```

## Technical Details

- Renders as `<footer>` element
- Full width by default
- Responsive padding
  ([Sprinkles Spacing]('/docs/guides/styles-and-theming/sprinkles')):
  - Mobile: 3
  - Tablet: 6
  - Desktop: 8
- Fixed minimum height (13,
  [Sprinkles Spacing]('/docs/guides/styles-and-theming/sprinkles'))
- Top border with theme color
- White background
- Centered alignment
- Best to place as the last child in the
  [GlobalLayout]('/docs/guides/components/globalLayout) component's children
  prop
- By design, it's not a fixed footer since fixed footers limit the viewport
- It's best to let the footer overflow in the page

## Semantic Guidelines

1. Global Level:

- The `<footer>` at the global level typically contains site-wide information,
  such as copyright notices, navigation links, or contact details.

2. Section Level:

- A `<footer>` inside a `<section>`, `<article>`, or `<aside>` represents footer
  content specific to that section, such as author information, related links,
  or a call to action.
