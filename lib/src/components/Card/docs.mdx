A polymorphic container component that provides a consistent visual structure
with predefined spacing, borders, and background color. The Card component is
designed to be flexible and can be rendered as any HTML element while
maintaining a cohesive look and feel.

## Usage

Import the Card component from the library:

```tsx
import { Card } from '@KyleWiteck/witeck-design/components';

// Basic usage
<Card>Content goes here</Card>;
```

## Basic Example

```tsx live
<Card>
  <Text>This is a basic card with default styling</Text>
</Card>
```

## As Different Elements

The Card component can be rendered as different HTML elements:

```tsx live
<Flex direction="column" gap="4">
  <Card element="article">
    <Text>This card is an article element</Text>
  </Card>

  <Card element="section">
    <Text>This card is a section element</Text>
  </Card>

  <Card element="aside" backgroundColor="neutral200">
    <Text>This card is an aside with custom background</Text>
  </Card>
</Flex>
```

## Custom Styling

Customize the card appearance using style props:

```tsx live
<Flex gap="4">
  <Card backgroundColor="primary100" boxShadow="base" width="200">
    <Text>Custom background and shadow</Text>
  </Card>

  <Card borderColor="primary" width="200">
    <Text>Custom border</Text>
  </Card>
</Flex>
```

## With Rich Content

## Responsive Layout

```tsx live
<Card
  // maxWidth="full"
  width={{
    mobile: 'full',
    tablet: '80',
    desktop: '100'
  }}
  padding={{
    mobile: '4',
    tablet: '6'
  }}
>
  <Text>This card adapts to different screen sizes</Text>
</Card>
```

## Best Practices

- Use consistent spacing within cards across your application
- Consider the semantic meaning when choosing the element type
- Maintain proper content hierarchy within cards
- Use appropriate padding for different screen sizes. Cards default padding is
  standard.
- Keep card content focused and related
- Consider loading states for dynamic content

## Accessibility

The Card component supports accessibility features through:

- Polymorphic element support for semantic HTML
- Proper spacing for readability
- Adequate color contrast
- Support for ARIA attributes
- Keyboard interaction support when interactive

## Technical Considerations

- Automatically handles border-radius consistency
- Memoized for performance
- Supports forwarded refs
- Default margin handling with `marginX="auto"`
- Flexible width handling with `boxSize="fitContent"`
  - `boxSize` enforces `height` and `width` to be the same
- Maintains consistent padding with default values
  ([Sprinkles Spacing](/docs/guides/styles-and-theming/sprinkles#spacing)):
  - Vertical padding: 6
  - Horizontal padding: 8
- Borders use theme-consistent colors
- Works with the application's spacing scale
