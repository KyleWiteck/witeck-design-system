Provides a **hover-based pop-up message** for additional context. It is built
using **Radix UI**, ensuring accessibility, smooth animations, and custom
styling.

It supports:

- Customizable colors (light or dark themes)
- Delay before appearing (200ms default)
- Automatic positioning and portal rendering

## How to Use

### Basic Example

When hovering over the **button**, the tooltip appears.

```tsx live
<Tooltip content="This is a tooltip" />
```

### Custom Trigger

If no `children` are provided, it defaults to an **Info Icon**.

```tsx live
<Tooltip content="Info tooltip">
  <Text>Hover me</Text>
</Tooltip>
```

## Accessibility & Behavior

- **Keyboard-friendly**: Works with focusable elements.
- **Delayed display**: Appears after **200ms** for better UX.
- **ARIA role**: Includes `role="tooltip"` for screen readers.

## Best Practices

- Use for **short, helpful messages** (avoid long descriptions).
- Avoid tooltips on **clickable** elements unless necessary.
