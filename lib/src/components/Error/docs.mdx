Error components provide a structured way to display error messages with a
headline, optional message, image, and configurable action buttons. They are
designed for both basic and advanced use cases, supporting two layout variants:

- `base`: A larger, centered layout for major error states.
- `small`: A compact, inline version for minor errors.

To streamline development, predefined error components handle common scenarios
such as "Not Found" or "Forbidden Access."

## How to Use

Each predefined error component comes with a default headline, message, and
image, reducing the need for manual setup.

- **NotFoundUI**: Displays "No Data Found" with a default message and a missing
  records image.
- **ForbiddenUI**: Displays "Forbidden Access", indicating a user lacks
  permissions.
- **NotAuthenticatedUI**: Displays "Authentication Required" when a user is not
  logged in.
- **UnknownErrorUI**: Displays a generic error message for unexpected failures.

### Import

```tsx
import {
  ForbiddenUI,
  NotAuthenticatedUI,
  NotFoundUI,
  UnknownErrorUI
} from '@peopleticker/magnit-design/components';
```

### Base variant

```tsx live
<Card>
  <NotFoundUI />
</Card>
```

### Small varinnt

```tsx live
<Card>
  <UnknownErrorUI
    variant="small"
    primaryAction={props => <Button {...props}>Go back</Button>}
  />
</Card>
```
